# 团队分工与角色（面向后续 Agent 协作）

为高效推进“剧情推动 + 养成进化 + 世界观扩展”，建议划分以下专业角色，允许多名Agent并行：

1. 前端演出工程师（FE-Show）
- 负责：文字演出流（自动滚动/分页）、面板交互、氛围层联动、事件委托
- 交付：local-game.html 增强、体验指标（响应<1s）

2. 大模型接入工程师（LLM-Integr)
- 负责：即梦4.0（vim provider）接入；提示词模板拆分与参数化；响应结构规范
- 交付：/api/story/next、/api/adventure/text-event、/api/story-agent/preview 返回 { text, deltas, meta }

3. 养成进化系统工程师（Evo-Engine)
- 负责：ThreeLayerEngine（L1/L2/L3）与固定算法融合；候选生成与决策固化
- 交付：/api/progress/apply 输出 { eligible, candidates[], text }；数值边界与平衡表

4. 世界观与叙事设计（Lore-Design)
- 负责：区域/族群/事件/角色/NPC任务模板；术语词典与风格审校
- 交付：docs/世界观蓝图.md、docs/提示词模板.md、术语表

5. 后端平台与运维（BE-Platform)
- 负责：路由稳定、日志与监控、降级策略；CDN/缓存与图片生成
- 交付：接口健康检查、错误码规范、部署脚本

6. 产品与体验统筹（PM-UX)
- 负责：里程碑计划、评审节奏、体验校准；需求变更管理
- 交付：迭代计划、评审记录、验收清单

协作规范
- 接口结构统一：{ text, deltas, meta, candidates?, outcome? }
- 文档入口统一：产品总览、迭代进度、世界观蓝图、提示词模板
- 失败降级：统一 showToast 与本地模板回退；日志标注模块来源
